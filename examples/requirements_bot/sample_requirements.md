# 요구사항 명세서 — 간단한 웹사이트 제작

## 1. 프로젝트 개요

프로젝트명: SimpleWeb
목적: 개인 포트폴리오 및 소개 페이지 제공
대상 사용자: 채용 담당자, 협업 제안자

---

## 2. 기능 요구사항

### 2.1 메인 페이지 (홈)
- 이름, 직함, 한 줄 소개 텍스트를 표시한다
- 상단 네비게이션 바에 홈, 소개, 프로젝트, 연락처 메뉴가 있다
- 네비게이션 메뉴 클릭 시 해당 섹션으로 스크롤 이동한다

### 2.2 소개 섹션
- 프로필 사진을 표시한다
- 간단한 자기소개 문단을 표시한다
- 보유 기술 스택을 태그 형태로 나열한다 (예: Python, React, SQL)

### 2.3 프로젝트 섹션
- 프로젝트 카드 목록을 표시한다
- 각 카드에는 프로젝트명, 설명, 사용 기술, GitHub 링크가 포함된다
- 카드는 최소 3개 이상 등록 가능해야 한다

### 2.4 연락처 섹션
- 이름, 이메일, 메시지 입력 폼을 제공한다
- 전송 버튼 클릭 시 입력값 유효성 검사를 수행한다
  - 이름: 필수, 1자 이상
  - 이메일: 필수, 이메일 형식 검증
  - 메시지: 필수, 10자 이상
- 유효성 검사 통과 시 이메일 전송 또는 콘솔 출력으로 처리한다

---

## 3. 비기능 요구사항

### 3.1 반응형 디자인
- 데스크탑(1280px 이상), 태블릿(768px~1279px), 모바일(767px 이하) 3가지 뷰를 지원한다
- 모바일에서 네비게이션 바는 햄버거 메뉴로 전환된다

### 3.2 성능
- 첫 페이지 로드 시간은 3초 이내여야 한다
- 이미지는 WebP 포맷을 사용하고 lazy loading을 적용한다

### 3.3 접근성
- 모든 이미지에 alt 텍스트를 제공한다
- 키보드 네비게이션을 지원한다

---

## 4. 기술 스택 제안

- 프론트엔드: HTML5, CSS3, JavaScript (또는 React)
- 스타일링: Tailwind CSS 또는 CSS Modules
- 배포: GitHub Pages 또는 Vercel

---

## 5. 제약 사항

- 백엔드 서버 없이 정적 파일로 동작해야 한다
- 외부 라이브러리는 CDN 또는 npm 패키지로만 사용한다
- 브라우저 지원 범위: Chrome, Safari, Firefox 최신 버전

---

## 6. 미결 사항

- 연락처 폼의 실제 이메일 전송 방식 결정 필요 (EmailJS, Formspree 등)
- 다국어 지원 여부 미정 (한국어 단일 or 한/영 전환)
- 다크모드 지원 여부 미정
